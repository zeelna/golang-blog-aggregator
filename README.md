# gator — Go RSS Blog Aggregator

A Go CLI that aggregates RSS feeds into a Postgres database. Supports registering/login, adding feeds, following/unfollowing feeds, periodic scraping, and browsing posts.

## Prerequisites

- **Go**: installed on your machine
  - In our environment: `go1.26.4`
- **Postgres**: installed and running
  - In our environment: `PostgreSQL 16.14`

> This repo uses a statically compiled Go CLI binary. After `go install`, you can run `gator` without needing the Go toolchain.

## Install Postgres (follow these steps)

Run the commands below (and keep an eye on prompts/output).

### 1) Install PostgresSQL
```bash
sudo apt update
sudo apt install postgresql postgresql-contrib
```

### 2) Confirm Postgres version is 15+
```bash
psql --version
```

### 3) Set the postgres user password
```bash
sudo passwd postgres
```

### 4) Enter the psql shell as the postgres OS user
```bash
sudo -u postgres psql
```

### 5) Create the database
```sql
CREATE DATABASE gator;
\c gator
```

At this point, your prompt should look like:
```text
gator=#
```

### 6) Set the user password inside Postgres (example)
```sql
ALTER USER postgres PASSWORD 'postgres';
```

### 7) Verify the Postgres server version
```sql
SELECT version();
```

### 8) Connect using connection string
```bash
psql "postgres://postgres:postgres@localhost:5432/gator"
```

### 8) Optional (if the password/database needs to be created again)
```bash
sudo -u postgres psql

ALTER USER postgres PASSWORD 'postgres';
CREATE DATABASE gator;
\q

psql "postgres://postgres:postgres@localhost:5432/gator"
```

## Database migrations
Goose provides feature to restore database schema to a previous state with 'goose ... down', by running migrations:
```bash
goose -dir sql/schema/ postgres postgres://postgres:postgres@localhost:5432/gator down
```

And to update the schema to a new state with 'goose ... up' 
```bash
goose -dir sql/schema/ postgres postgres://postgres:postgres@localhost:5432/gator up
```

### Configuration
The program reads configuration from a file in your home (and not project's root) directory:
```text
 ~/.gatorconfig.json
```
This file is excluded from your git repo (it contains DB credentials).

Create it:
```bash
nano ~/.gatorconfig.json
```

Example content (use your exact local values):
```json
{
  "db_url": "postgres://postgres:postgres@localhost:5432/gator?sslmode=disable",
  "current_user_name": ""
}
```

## Install SQLc
SQLC is an amazing Go program that generates Go code from SQL queries. It's not exactly an ORM, but rather a tool that makes working with raw SQL easy and type-safe.
[SQLC postgres docs](https://docs.sqlc.dev/en/latest/tutorials/getting-started-postgresql.html)

We will use Goose to manage our database migrations (the schema), and SQLC to generate Go code that our application can use to interact with the database (run queries).

SQLC is just a command line tool, it's not a package that we need to import. I recommend installing it using go install.
Installing Go CLI tools with go install is easy and ensures compatibility with your Go environment.

You'll notice that the generated code relies on Google's uuid package, so you'll need to add that to your module:
```bash
go install github.com/sqlc-dev/sqlc/cmd/sqlc@latest
```

### 1) SQLC requires Google UUID 
SQLC generated Go code in internal/database with command 'sqlc generate'. You'll notice that the generated code relies on Google's uuid package, so you'll need to add that to your module:
```bash
go get github.com/google/uuid
```

### 2) SQLC requires Postgres Driver
We need to add and import a Postgres driver so our program knows how to talk to the database. Install it in your module:
```bash
go get github.com/lib/pq
```
### 3) SQLC must import Postgres Driver
Add this import to the top of your main.go file:
```text
import _ "github.com/lib/pq"
```

## SQLC Configuration (Optional) - Update Database schema or queries
Whenever you create a new database query in project_root/sql/queries, or a new database schema updates in project_root/sql/schema, you must run the following command with SQLc:
```bash
sqlc generate
```

### SQLc configuration to generate Go code for Database operations
´´´bash
cat ./sqlc.yaml
´´´

The YAML configuration for SQLc specifies the location to store the code generated by SQLc
```yaml
version: "2"
sql:
  - schema: "sql/schema"
    queries: "sql/queries"
    engine: "postgresql"
    gen:
      go:
        out: "internal/database"
```

## Usage
### 1) Must: Start a Background Process
Open a new Terminal process to read new RSS blog posts, requesting updates between specified time duration - the blog aggregation functionality. 

#### Fetch RSS Feed, updating /scraping all the previosuly added feeds (with command 'addfeed') of new posts. This occurs each time-interval passes, specfied as the CLI argument (1s, 3m30s or 2h):
```bash
gator agg 1m
```

This aggregation draws RSS feed updates whenever 1 minute passes and stores in the database table named 'posts'. 
You can change the internal parameter to 1s (1 second), 1m (1 minute), 1h (1 hour) or any specified time in that format how often blog updates are read.

### 2) Main Process to interact with RSS Feeds
After installing and migrating, you can run the CLI like:
```bash
gator <command> [args...]
```

#### 2.1) Register a user
Register a new user that will interact with RSS Feeds, specify their name and save them in the database table 'users':
```bash
gator register alice
```

#### 2.2) Login a user
Log in the specified user, if they exist in the database table 'users':
```bash
gator login alice
```

#### 2.3) Display all users
Display all registered users in the database table 'users' that will be interacting with feeds: 
```bash
gator users
```

#### 2.4) Delete all users
Delete all users that were registered using command 'gator register'. This deletes all entries in the 'users' database table.
```bash
gator reset
```

#### 2.5) Add a new RSS Feed with URL
Save a new RSS Feeds to the database that users can interact with:
```bash
gator addfeed "Hacker News" "https://hnrss.org/newest"
gator addfeed "Lanes Blog" "https://www.wagslane.dev/index.xml"
```

#### 2.6) Display all RSS Feeds
Display all the RSS Feeds the CLI command like has saved:
```bash
gator feeds
```

#### 2.7) Follow as RSS Feed (as logged in user)
Currently logged in user follows a specified RSS Feed. The logged-in user is stated in '~/.gatorconfig.json'
```bash
gator follow "https://hnrss.org/newest"
gator follow "https://www.wagslane.dev/index.xml"
```

#### 2.8) Display all RSS Feeds user follows
Display all the RSS Feeds current logged in user is following
```bash
gator following
```

#### 2.9) Unfollow user from a RSS Feed
Unfollow the user from a RSS Feed via its URL:
```bash
gator unfollow "https://hnrss.org/newest"
```

#### 2.10) Browse newest posts from RSS Feeds
Browse the newest posts coming from RSS Feeds, displaying the maximum limit of 10 posts, or any specified number (default limit is 2):
```bash
gator browse 10
```

### Database
The schema includes 4 core tables:
```text
    users
    feeds
    feed_follows (join table for many-to-many)
    posts
```
